# Lumno 0.9.8 重构计划

## 权威基线

- 本次重构从 Chrome Web Store 运行包恢复，而不是从 `0.9.8` tag 直接恢复。
- 权威包路径：`dist/lumno-store-v0.9.8.zip`。
- 该包内 `manifest.json` 的版本是 `0.9.8`。
- 当前仓库里的 `0.9.8` tag 对应的 `manifest.json` 仍是 `0.9.7`，因此只能作为历史参考，不能作为本次恢复基线。
- `search-rule-changes-since-571b607.md` 是迁移材料，恢复基线时必须保留。

## 目标

- 功能和交互与 0.9.8 商店包保持一致。
- 样式视觉与 0.9.8 商店包保持一致。
- 优化普通网页浮层加载性能，尤其降低翻译软件或其他扩展对 Lumno 浮层的干扰。
- 将大文件拆成可维护结构，减少重复逻辑和补丁式分支。
- 在结构稳定后，基于 `search-rule-changes-since-571b607.md` 的思想，以更合适的共享实现应用搜索/provider 改进。

## `!important` 处理原则

- 不直接批量删除 `!important`。
- 每个 `!important` 先归类：
  - 宿主页隔离必要：外层浮层定位、z-index、几何同步、进入/退出动画保护。
  - 历史补丁遗留：自有页面、shadow root 内部、可由类名/data-state/CSS 变量表达的状态样式。
  - 动态测量必要：由 JS 计算宽高、位置、padding 的实时几何值。
- 删除或替换前必须先记录现有视觉和交互依赖。
- 对 `newtab.html`、`newtab.js`、`options.html`、`options.js`，优先用稳定类名、CSS 变量和 data-state 替代。
- 对注入路径，先通过 shadow root 和自有样式根缩小受污染面，再判断是否能移除 `!important`。

## 阶段计划

### 1. 恢复 0.9.8 运行基线

- 新建工作分支。
- 从 `dist/lumno-store-v0.9.8.zip` 解出运行文件并覆盖仓库对应文件。
- 保留开发材料和迁移文档。
- 跑语法检查和打包内容检查，确认恢复后的运行包和权威包可对齐。

### 2. 建立重构契约

- 记录 manifest 权限、content scripts、web accessible resources。
- 记录 overlay/newtab/options 的关键 DOM 节点、类名、data-state、CSS 变量和动态样式入口。
- 记录当前 `!important` 热点和用途。
- 建立可自动跑的基础检查脚本，至少覆盖：
  - JS 语法。
  - manifest 引用资源存在。
  - store zip 白名单内容。
  - 搜索/provider fixtures。

### 3. 只拆结构，不改行为

- 保持根入口文件名不变，避免破坏 Chrome 扩展加载。
- 将共享逻辑拆到明确模块：
  - `src/shared/`：通用常量、URL、storage、i18n、provider 归一化。
  - `src/search/`：query context、匹配、打分、候选合并、最终排序。
  - `src/overlay/`：普通网页浮层 shell、shadow root、渲染、事件。
  - `src/newtab/`：newtab 状态、布局、建议渲染。
  - `src/options/`：设置页状态、provider 编辑、快捷键/黑名单 UI。
- 初期优先使用简单构建或生成脚本，不引入复杂依赖。

### 4. 浮层隔离和加载性能

- `hotkey-listener.js` 保持轻量，只处理快捷键、消息和最小 toast。
- 普通网页浮层改成整块隔离，输入区、结果区、tooltip、prefix、按钮、空状态、AI 装饰层都进入扩展自有样式根。
- 对浮层根和关键节点统一加 `translate="no"`、`data-no-translate="true"`、`notranslate`。
- MutationObserver 只在需要时启动，并且具备退避和停止条件。
- 减少首帧前 DOM 写入和多层 requestAnimationFrame 链条，避免和翻译扩展抢 DOM。

### 5. 有验证地治理 `!important`

- 先保留所有外层几何相关 `!important`。
- 从自有页面和 shadow root 内部开始替换。
- 每一组替换都要对照样式契约和浏览器截图/交互验证。
- 不能验证样式一致时，暂时不删。

### 6. 重新应用搜索/provider 改进

- 不直接 cherry-pick 后续 UI 提交。
- 普通搜索质量吸收 `3761037` 的思想：
  - token/host/path/pinyin 匹配。
  - 短 ASCII 宽松匹配收紧。
  - coverage-aware scoring。
  - candidate pool、dedupe、host diversity、blacklist 统一管线。
- AI/site-search provider 吸收 `0a94ca5` 的思想：
  - `openAndSubmit` 和无 `{query}` template 统一识别为 AI provider。
  - 自定义 provider 继承内置 provider 的 `action` 和 `submitStrategy`。
  - `background`、`newtab`、`options` 使用同一套 provider helper。
