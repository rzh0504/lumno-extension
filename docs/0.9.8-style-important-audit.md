## 风险分类

### 必须先保留

- 普通网页 overlay 外层几何：
  - fixed 定位。
  - `top` / `left` / `width` / `max-width` / `max-height`。
  - `z-index: 2147483647`。
  - 进入/退出动画中的 `transform` / `opacity` / `filter`。
  - page zoom / visualViewport 同步。
- 宿主页可能污染的 light DOM UI：
  - overlay suggestion item。
  - overlay tooltip。
  - overlay prefix。
  - overlay buttons。
  - overlay list container。
- `input-ui.js` 里的 `all: unset` reset，在迁入 shadow root 前不能直接删。

### 可以优先替换

- newtab 自有页面内的状态样式：
  - selected / hover / active。
  - display / visibility。
  - tag color。
  - history delete button 状态。
- newtab / options 静态页面里的 `!important`。
- shadow root 内部样式，一旦整块隔离后可以去掉大部分 `!important`。
- 通过 JS 反复写入的视觉状态，优先替换成 class/data-state。

### 需要验证后替换

- favicon optical alignment。
- suggestions container 显隐和高度。
- tooltip 定位和 transform。
- site-search prefix 左侧 padding、输入框 padding 同步。
- bottom dock 宽度和 body padding-top。

## 治理顺序

1. 先做结构拆分，不删视觉 important。
2. 建立 overlay shadow root，减少宿主页样式污染面。
3. 把 `input-ui.js` 迁为可在 shadow root 内工作的组件。
4. 把 newtab 动态状态从 `style.setProperty(..., 'important')` 改成 class/data-state。
5. 每组替换后跑自动检查和浏览器截图/交互验证。

## 已处理项

- `src/newtab/newtab.js`：删除 root 初始化阶段对 `padding`、`width`、`max-width`、`box-sizing` 的 inline important 写入，继续由既有页面 CSS 承接。
- `src/newtab/newtab.js`：删除 bottom dock width 的 inline important 写入，继续使用 `--x-nt-content-max-width`。
- `src/overlay/lifecycle.js`：仅迁移 overlay rAF、viewport/zoom 同步和 anti-translate observer 生命周期，不删除注入浮层 CSS important。
- `src/overlay/shell.js`：浮层内容迁入 shadow root，先减少宿主页 CSS 命中面；内部 visual important 暂不删除。
- `src/overlay/shell.js`：新增 light DOM host 的 fixed/z-index/display/visibility/opacity/transform/filter/overflow 等最小外壳 important，用于抵御宿主页或其他扩展对 host 的全局选择器污染。该 important 属于隔离外壳保护，不用于调整内部视觉。
- `src/overlay/shell.js`：删除 shadow stylesheet 内图标、placeholder、selection 小范围 `!important`；这些规则已经在 shadow root 内，且加载顺序晚于字体 stylesheet，普通层叠足够稳定。
- `src/overlay/shell.js`：删除 shadow stylesheet 内滚动条隐藏规则的 3 处 `!important`；规则位于 shadow root 内，普通层叠即可承接，不影响外层 host/panel 防污染边界。
- `src/shared/url-guards.js` + `src/background/background.js`：扩展商店/浏览器保留页不再进入 favicon HTML fetch 路径，减少后台 CORS 错误和无效加载。
- `src/options/options.html`：删除剩余 3 处静态 `!important`。快捷键 reset action 改用更高语义的 hover + data-state 选择器锁定隐藏状态；JS tooltip host 改用普通 `content: none` 关闭伪元素。
- `src/newtab/newtab.js`：删除 Logo 与 mode badge 这组自有页面静态 UI 的 inline important，改为普通 inline style；显示/隐藏和主题 opacity 仍由同一函数控制。
- `src/shared/search-utils.js`：强 navigation match 置顶从 `background.js` / `newtab.js` 两份复制逻辑收敛为共享函数，减少搜索交互分叉风险。
- `src/newtab/newtab.js`：删除 newtab 搜索 root 宽度、搜索输入组件、search layer、right icon、站内搜索 prefix、prefix/input padding 与 caret color 这组自有页面 important；继续依赖 `--x-nt-search-max-width`、普通 inline style 和稳定 DOM 层级承接。
