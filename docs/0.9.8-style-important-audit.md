# Lumno 0.9.8 `!important` 审计

## 基线统计

恢复自 `dist/lumno-store-v0.9.8.zip` 后，`!important` / `style.setProperty(..., 'important')` 热点如下：

| 文件 | 计数 | 主要用途 |
| --- | ---: | --- |
| `background.js` | 1191 | 普通网页浮层、suggestion item、tooltip、站内搜索 prefix、favicon、动画和几何同步 |
| `newtab.js` | 1016 | newtab 搜索框、suggestions、bookmark/recent dock、tooltip、favicon、动态状态 |
| `input-ui.js` | 107 | 搜索输入组件 reset、icon、right button、容器 |
| `options.js` | 0 | 暂无 JS important |
| `options.html` | 3 | 静态样式 |
| `newtab.html` | 1 | 静态样式 |

## 分布

`background.js` 中的热点集中在：

- `6501-7500`：overlay 容器、输入框、tooltip、按钮。
- `9001-9500`：站内搜索 prefix。
- `11001-12000`：open tabs / suggestion item 前半段。
- `12501-13500`：suggestion item 后半段、history delete、suggestions container、进入动画。

`newtab.js` 中的热点集中在：

- `1-500`：root、wordmark、notice banner。
- `4001-4500`：site-search prefix、suggestions container、tooltip、bottom dock。
- `8501-10000`：suggestion item、tag、history delete、hover/active 状态。
- `10501-11000`：mode badge、wordmark、search layer、site-search prefix。

## 风险分类

### 必须先保留

- 普通网页 overlay 外层几何：
  - fixed 定位。
  - `top` / `left` / `width` / `max-width` / `max-height`。
  - `z-index: 2147483647`。
  - 进入/退出动画中的 `transform` / `opacity` / `filter`。
  - page zoom / visualViewport 同步。
- 宿主页可能污染的 light DOM UI：
  - overlay suggestion item。
  - overlay tooltip。
  - overlay prefix。
  - overlay buttons。
  - overlay list container。
- `input-ui.js` 里的 `all: unset` reset，在迁入 shadow root 前不能直接删。

### 可以优先替换

- newtab 自有页面内的状态样式：
  - selected / hover / active。
  - display / visibility。
  - tag color。
  - history delete button 状态。
- newtab / options 静态页面里的 `!important`。
- shadow root 内部样式，一旦整块隔离后可以去掉大部分 `!important`。
- 通过 JS 反复写入的视觉状态，优先替换成 class/data-state。

### 需要验证后替换

- favicon optical alignment。
- suggestions container 显隐和高度。
- tooltip 定位和 transform。
- site-search prefix 左侧 padding、输入框 padding 同步。
- bottom dock 宽度和 body padding-top。

## 治理顺序

1. 先做结构拆分，不删视觉 important。
2. 建立 overlay shadow root，减少宿主页样式污染面。
3. 把 `input-ui.js` 迁为可在 shadow root 内工作的组件。
4. 把 newtab 动态状态从 `style.setProperty(..., 'important')` 改成 class/data-state。
5. 每组替换后跑自动检查和浏览器截图/交互验证。

## 已处理项

- `src/newtab/newtab.js`：删除 root 初始化阶段对 `padding`、`width`、`max-width`、`box-sizing` 的 inline important 写入，继续由既有页面 CSS 承接。
- `src/newtab/newtab.js`：删除 bottom dock width 的 inline important 写入，继续使用 `--x-nt-content-max-width`。
- `src/overlay/lifecycle.js`：仅迁移 overlay rAF、viewport/zoom 同步和 anti-translate observer 生命周期，不删除注入浮层 CSS important。

## 暂不处理项

- `src/background/background.js` 与 `src/overlay/input-ui.js` 中的浮层几何、reset、suggestion item、tooltip、prefix、按钮 important 暂时保留。
- 在整块浮层进入 shadow/self-owned style root 并完成浏览器视觉验证前，不继续削减注入路径 important。
